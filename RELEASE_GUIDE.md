# Aetherium Desktop Release Guide

## Overview
This guide covers the complete process of building, releasing, and distributing a new version of Aetherium Desktop with automatic update support.

## Prerequisites
- Node.js v18+ installed
- Git configured with GitHub access
- GitHub CLI (`gh`) installed and authenticated
- Windows development environment (for building Windows executables)

## Release Process

### 1. Update Version Number
1. Edit `package.json` and increment the version number:
   ```json
   "version": "1.2.12"  // Increment to 1.2.13, 1.3.0, etc.
   ```
2. Update the web app version in Supabase:
   ```bash
   curl -s -X POST "https://api.supabase.com/v1/projects/ejtqfbgwvcycmeezabyp/database/query" \
     -H "Authorization: Bearer $(cat ~/.config/supabase/access_token)" \
     -H "Content-Type: application/json" \
     -d '{"query": "UPDATE app_config SET value = '"'"'{\"web\": \"2.7.XXX\", \"desktop\": \"1.2.12\"}'"'"'::jsonb WHERE key = '"'"'versions'"'"'"}'
   ```

### 2. Commit and Push Changes
```bash
git add package.json main.js  # Add any changed files
git commit -m "Release v1.2.12: [Brief description of changes]"
git push origin master
```

### 3. Build the Executable

**⚠️ IMPORTANT: Must build from Windows PowerShell, NOT WSL!**

WSL2 Linux cannot create Windows executables. Use one of these methods:

#### Method 1: PowerShell Script (Recommended)
```powershell
# Open Windows PowerShell (not WSL!)
# Navigate to desktop folder:
cd C:\Users\WolfDrago12\Desktop\Aetherium\aetherium-desktop

# Run the build script:
.\build-windows.ps1
```

#### Method 2: Manual Build
```powershell
# Open Windows PowerShell
cd C:\Users\WolfDrago12\Desktop\Aetherium\aetherium-desktop

# Install dependencies (if needed)
npm install

# Build for Windows
npm run build:win
```

This will:
- Build the Electron app using electron-builder
- Create an installer at `dist/Aetherium-Setup.exe`
- Generate a `latest.yml` file for auto-updates

Build output location: `dist/`

**Build output files:**
- `dist/Aetherium-Setup.exe` - The installer users download
- `dist/latest.yml` - Auto-update configuration (must upload to GitHub release)
- `dist/win-unpacked/` - Unpacked app files (for testing, don't distribute)

### 4. Create GitHub Release
```bash
# Using GitHub CLI:
gh release create v1.2.12 \
  --title "Aetherium Desktop v1.2.12" \
  --notes "## What's New
- Fixed overlay positioning (top-left instead of top-right)
- Fixed overlay z-index to appear above main window
- [Add other changes here]

## Installation
Download and run Aetherium-Setup.exe below.

## Auto-Update
If you already have Aetherium Desktop installed, the app will automatically check for and download this update when you launch it." \
  dist/Aetherium-Setup.exe \
  dist/latest.yml

# Or manually via GitHub web interface:
# 1. Go to https://github.com/TheEtherWolf/aetherium-desktop/releases/new
# 2. Tag: v1.2.12
# 3. Title: Aetherium Desktop v1.2.12
# 4. Upload: dist/Aetherium-Setup.exe and dist/latest.yml
# 5. Publish release
```

### 5. Verify Auto-Update
1. Install the previous version on a test machine
2. Launch the app
3. The update notification should appear automatically
4. Click "Download Update" → app downloads and prompts to restart
5. Restart → app updates to new version

## Auto-Update Architecture

### How It Works
1. **Check for Updates**: On app launch, electron-updater checks GitHub releases for a `latest.yml` file
2. **Compare Versions**: Compares the version in `latest.yml` with the currently installed version
3. **Download**: If newer version exists, downloads the installer in the background
4. **Install**: User clicks "Restart & Update" → app closes → installer runs → app reopens with new version

### Key Files
- `latest.yml`: Auto-generated by electron-builder, contains download URL and version info
- `main.js`: Contains auto-update logic (lines ~45-100)
- `update-window.html`: Update notification UI

### Configuration
Auto-update is configured in `package.json`:
```json
"build": {
  "publish": {
    "provider": "github",
    "owner": "TheEtherWolf",
    "repo": "aetherium-desktop"
  }
}
```

## Troubleshooting

### Build fails with "command not found: electron-builder"
```bash
npm install
```

### Auto-update not working
1. Verify `latest.yml` is uploaded to GitHub release
2. Check that version in `latest.yml` is higher than installed version
3. Check DevTools console for update errors (press Ctrl+Shift+I in app)

### Installer not running
- Windows SmartScreen may block unsigned executables
- User must click "More info" → "Run anyway"
- (Future: Consider code signing certificate to avoid this)

## Version Numbering
Follow semantic versioning (MAJOR.MINOR.PATCH):
- **MAJOR** (1.x.x): Breaking changes, major redesigns
- **MINOR** (x.2.x): New features, improvements
- **PATCH** (x.x.12): Bug fixes, small tweaks
